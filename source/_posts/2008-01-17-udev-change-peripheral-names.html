--- layout: post title: udev change peripheral names tags: - debian - eth0 - ethernet - Linux - ubuntu - udev - vmware status: publish type: post published: true meta: {} --- Often, in VMware Linux guests, it happens that the mac address of the virtual Ethernet card changes due to various operations I wont list here and, like in physical machines when you change the network card, the default Ethernet name changes from eth0 to eth1. If you want to restore the eth0 name, just edit the udev rule like this: ``` sudo nano /etc/udev/rules.d/70-persistent-net.rules ``` and change eth1 to eth0 minding to delete the row relative to the previous card (virtual or real) . That's it!
